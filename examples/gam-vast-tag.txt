# The Nexus Engine - Catalyst GAM VAST Tag (Publisher Enrichments)
#
# This template lives in the repo. The Nexus Engine team bakes in our own
# TCF vendor (GVL) ID before emailing the tag to a publisher; the publisher
# only fills in the publisher-side placeholders below and pastes the URL
# into Google Ad Manager as a "Third Party VAST Redirect" creative. GAM
# substitutes every %%...%% macro at ad call time and Catalyst maps them
# onto OpenRTB 2.5 fields before calling demand.
#
# Nexus Engine fills in before delivery:
#   {GVL_ID}        -> our registered IAB TCF v2 vendor ID (numeric).
#                      Required - GAM returns empty if the GVL is unknown.
#
# Publisher fills in when trafficking:
#   {PUBLISHER_ID}  -> Nexus Engine publisher ID (e.g. pub-video-123456)
#   {MIN} / {MAX}   -> allowed creative duration in seconds (e.g. 5 / 30)
#   {FLOOR}         -> USD bid floor (omit bidfloor= to disable)
#
# Endpoint: ads.thenexusengine.com is the live Catalyst host (no separate
# test subdomain - staging runs behind the same hostname via nginx traffic
# split). Use a test publisher ID during the initial 1,000-request window
# and switch to production once validation passes.
#
# ---------------------------------------------------------------------------

# ---------------------------------------------------------------------------
# OpenRTB 2.5 mapping (reference)
#
#   page_url        -> site.page
#   domain          -> site.domain
#   ref             -> site.ref
#   description_url -> site.content.url
#   placement_id    -> imp[0].tagid
#   w / h           -> imp[0].video.w / imp[0].video.h
#   sport           -> site.content.genre + imp.ext.context.sport
#   competition     -> site.content.series + imp.ext.context.competition
#   lang            -> site.content.language (BCP-47)
#   device_type     -> device.devicetype (OpenRTB enum)
#   geo             -> device.geo.country (ISO 3166-1 alpha-3)
#   content_type    -> site.content.cattax + site.content.cat[]
#   gdpr            -> regs.ext.gdpr
#   gdpr_consent    -> user.ext.consent
#   addtl_consent   -> user.ext.ConsentedProvidersSettings.consented_providers
#   us_privacy      -> regs.ext.us_privacy
#   gpp / gpp_sid   -> regs.gpp / regs.gpp_sid[]
#   coppa           -> regs.coppa
#   lmt             -> device.lmt
#   ifa / ifa_type  -> device.ifa / device.ext.ifa_type
#   ua              -> device.ua
#   schain          -> source.ext.schain
#
# For first-party identifiers (UID2, ID5, LiveRamp, SharedID, PPID) switch to
# POST /video/openrtb and populate user.ext.eids[] - the GET tag cannot carry
# structured EID arrays.
