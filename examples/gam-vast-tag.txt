# The Nexus Engine - Catalyst GAM VAST Tag (Publisher Enrichments)
#
# Two-layer substitution:
#   1. GAM macros (%%...%%)    -> resolved by the publisher's Google Ad
#                                 Manager instance as it emits the VAST
#                                 Wrapper redirecting to us.
#   2. IAB VAST 4 macros ([...]) -> resolved by the video player (e.g.
#                                   Dailymotion) as it chases the wrapper
#                                   AdTagURI per IAB VAST 4.2 spec.
# Both passes happen before the request hits Catalyst, so every %%...%% and
# [...] arrives populated.
#
# {GVL_ID} is our own registered IAB TCF vendor ID - a static constant,
# baked into the tag once at template level. Publishers never see it; when
# a CMP on the publisher's page has us in its vendor list, GAM emits the
# full signed TCF consent string via %%GDPR_CONSENT_{GVL_ID}%%. Catalyst
# decodes that string server-side and applies per-bidder TCF GVL checks
# before fanning out to SSPs - no further publisher action required.
#
# Before emailing a copy of this URL to a publisher:
#   {GVL_ID}   -> replace with our numeric GVL ID (one-time, repo-wide).
#
# Publisher fills in when trafficking:
#   {PUBLISHER_ID}  -> Nexus Engine publisher ID (e.g. pub-video-123456)
#   {MIN} / {MAX}   -> allowed creative duration in seconds (e.g. 5 / 30)
#   {FLOOR}         -> USD bid floor (omit bidfloor= to disable)
#
# Endpoint: ads.thenexusengine.com is the live Catalyst host (no separate
# test subdomain - staging runs behind the same hostname via nginx traffic
# split). Use a test publisher ID during the initial 1,000-request window
# and switch to production once validation passes.
#
# ---------------------------------------------------------------------------

# OpenRTB 2.5 mapping (reference)
#
# ------ Layer 1: GAM macros (substituted by Google Ad Manager) -------------
#
# VAST Enrichments (standard macros)
#   page_url        -> site.page
#   domain          -> site.domain
#   ref             -> site.ref
#   cb              -> (not carried; forces fresh fetch)
#   placement_id    -> imp[0].tagid                          (Ad unit path)
#   w / h           -> imp[0].video.w / imp[0].video.h       (slot size)
#
# Custom Key-Values (contextual targeting)
#   sport           -> site.content.genre  + imp.ext.context.sport
#   competition     -> site.content.series + imp.ext.context.competition
#   lang            -> site.content.language (BCP-47)
#   device_type     -> device.devicetype (OpenRTB enum)
#   geo             -> device.geo.country (ISO 3166-1 alpha-3)
#   content_type    -> site.content.cattax + site.content.cat[]
#
# Privacy Signals (GAM native, passed through where applicable)
#   gdpr            -> regs.ext.gdpr
#   gdpr_consent    -> user.ext.consent
#   addtl_consent   -> user.ext.ConsentedProvidersSettings.consented_providers
#   us_privacy      -> regs.ext.us_privacy
#   gpp / gpp_sid   -> regs.gpp / regs.gpp_sid[]
#   coppa           -> regs.coppa
#
# User Signals (privacy-compliant identifiers via GAM)
#   ifa / ifa_type  -> device.ifa / device.ext.ifa_type
#   lmt             -> device.lmt
#
# ------ Layer 2: IAB VAST 4 macros (substituted by the player) -------------
# Dailymotion's vast-client-js resolves these per IAB VAST 4.2 spec when
# the player chases our wrapper redirect. Any unsupported macro is emitted
# as "-1" per spec - Catalyst treats "-1" as null on the receive side.
#
# Content (net-new vs GAM)
#   content_id          -> site.content.id           (Dailymotion video ID)
#   content_url         -> site.content.url          (content asset URL)
#
# Player / Session
#   player_size         -> imp[0].video.w / .h       (actual player dims; prefer over slot w/h)
#   inventory_state     -> imp[0].ext.inventorystate (autoplay/muted bitmask)
#   placement_type      -> imp[0].video.placement    (IAB placement enum)
#   api                 -> imp[0].video.api[]        (VPAID/OMID/MRAID)
#   omid_partner        -> imp[0].video.ext.omidpartner
#   verification_vendors -> imp[0].ext.verification
#   transaction_id      -> bidrequest.id             (per-request unique)
#
# App / CTV
#   app_bundle          -> app.bundle                (populated for in-app/CTV; empty on web)
#
# Geo precision
#   lat_long            -> device.geo.lat / .lon     (consent-gated; sharpens geo=)
#
# Resolution rules when both layers contribute to the same OpenRTB field:
#   imp.video.w/h         -> player_size wins when present, else GAM w/h
#   imp.video.placement   -> placement_type wins when present, else hardcoded placement=1
#   device.geo.*          -> GAM geo (country) + player lat_long (precision), merged
#
# NOT in this tag (per publisher scope):
#   - Deterministic first-party IDs (UID2 / ID5 / LiveRamp / SharedID / PPID).
#     When the publisher's 1p strategy is ready, switch that line item to
#     POST /video/openrtb and populate user.ext.eids[] - the GET tag cannot
#     carry structured EID arrays.
#   - supply chain object (source.ext.schain) - handled server-side by
#     Catalyst, not injected from the publisher side.
#   - Duplicate IAB macros that GAM already provides ([GDPRCONSENT], [IFA],
#     [IFATYPE], [LIMITADTRACKING], [PAGEURL], [DOMAIN], [CACHEBUSTING],
#     [DEVICEUA], [CLIENTUA]) - dropped to avoid two-source-of-truth drift.
